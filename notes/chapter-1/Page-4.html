# 无限白嫖Onedrive

以前的Office 365开发者试用订阅（简称E3）有效期只有一年，一年过后数据都没了。
现在微软推出的新版订阅（E5）有效时间为90天，可通过订阅来增加时间。

  1. # 加入计划

    首先你需要有一个微软账号，账号注册就不多说了。
    我们接着进入到Microsoft365官网：https://developer.microsoft.com/zh-cn/office/dev-program (opens new window)

    登录你的微软账号并按照提示填写信息

  2. # 创建订阅

    需要注意的是，这里选择的国家很重要，关系到你的onedrive数据等存储位置。比如我选择US，那么我的数据就是放在美国的微软数据中心。 下面张图中的“域名”弄错了，因该是 “域”,可随意填写

    填写手机号进行验证

    这样就算是注册完成了

  3. # 添加许可证

    对自己的账号添加许可证，不然用不了。

    点击上图的 Go to subscription ，进入后台 管理 - > 用户 -> 活跃用户 -> 点击用户名 -> 许可证和用户 ,添加好后点击应用就可以了

  4. # 修改OneDrive容量

    修改默认的onedrive 1024 为 5120
    现在直接在后台修改即可：https://admin.microsoft.com/Adminportal/Home?source=applauncher#/users (opens new window)

  5. # 自动订阅

    # 注册自己的api key

    登录进入 azure (opens new window) ，登录账号使用你的e5账户 ，就是以xxx.onmicrosoft.com开头的的账户。
    进入https://portal.azure.com/#blade/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/RegisteredApps
    点击 新注册

    名称随意取一个，但是最重要的是 “重定向 URI (可选)”，请填写为下列地址，不然程序收不到回调。

    https://e5.qyi.io/outlook/auth2/receive
    1

    # 获取ID与密钥

    点击注册后记录以下信息: 1、应用程序(客户端)ID

    2、客户端密码
    这里截止时间目前已经变更，选择可选的最长时间即可，到时间需要更换

    很多同学反应授权报错 Invalid client secret is provided. 这里请注意了，客户端密码请复制 “值”，不要复制 ID。

    将这两个信息保存下来一会会用到

    # 配置api权限

    勾选一下四个选项后，同时点击 代表XX授予管理员同意 PS: 这一步如果用的是子账号创建的api，那么这一项是灰色的，不能点击。需要登录 管理员 账号，再点击 代表XX授予管理员同意

    这个时候api的配置就算完成了

    # 添加到自动订阅

    进入 https://e5.qyi.io/ 这里需要github账户登录，同样的，我只能获取你在 github中的 用户id、用户名等基础信息(邮箱获取不了)，其他的任何信息也获取不到(可自行尝试注册一个github Apps测试)。

    点击 新建 ，名称随意输入，只是个标识而已

    填入上一步记录的 应用程序(客户端)ID、客户端密码 client_id ->应用程序(客户端)ID client_secret->客户端密码

    点击下一步进行配置调用时间，一般默认即可。
    说明：单位 秒（最低调用频率为 60 秒,最高为6小时），例如: 30-60,代表在30秒-60秒之间随机调用一次

    最后，点击取授权，此时会跳转到 microsoftonline Auth2.0授权页面，在这里请注意使用同一个域下的空账号(子账号)进行授权。 (因为在这一步我能获取到授权的outlook账户 邮件，但是程序不会保存，仅仅是调用api。)以免在以后发生误会。

    点击 接受 后，将会跳转回自动订阅程序。

    点击 接受 后，如果出现报错，请先检查 client_id 和 client_secret是否正确填写。

    至此，你就获得了一个理论可以永久使用的Mcirosoft365 E5账号。
