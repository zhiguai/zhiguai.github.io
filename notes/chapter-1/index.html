# 前言

# 项目介绍

搭建一个自己的多功能云盘。
如阿里云盘 (opens new window)，百度云盘 (opens new window)那样。同时，可以让您快速搭建起公私兼备的网盘系统。 在底层支持不同的云存储平台，用户在实际使用时无须关心物理存储方式。可以运用于，个人用网盘、文件分享系统，亦或是针对大小团体的公有云系统和私密的加用NAS云。

# 实现方案

材料 教程所选 作用 优点
1.网盘程序 (基于Docker)Cloudreve (opens new window) 提供核心服务 ...
存储方案 onedrive 实际存储服务器 容量大且流量可直走onedrive服务器
2.离线下载 (基于Docker)Aria2 (opens new window) 对接网盘离线下载服务 ...
3.底层服务 (宝塔管理)Docker (opens new window) 方便快捷且安全的运行上层服务 使用方便，部署快捷，可自动选择适应当前依托硬件的ARM,X86，等等架构。
4.内网穿透 (宝塔管理)网云穿 (opens new window) 将网盘映射出去实非内网访问 有一条1M免费隧道，且配套软件使用方便
5.反向代理 (宝塔管理)NGINX (opens new window) 获得更丰富的WEB服务的扩展功能 ...
6.运维面板 宝塔面板 (opens new window) 提供方便的图形运维界面 对于小白来说真是方便
7.操做系统 linux (opens new window) 基础 ...
8.依托硬件 树莓派3B (opens new window) 基础(4h1g.RAM.千兆网口) 作者手中闲置的一块

注意

  • 上述实现方案的核心就是Cloudreve所以其他“材料”可以按照自己的情况更换，可在同类产品中找到替代的，就比如说，树莓派3B+或4B以及其他装有与教程相同的“操作系统”的云服务器同样可以适用于本笔记的教程。。
  • 教程所采用的方案是作者认为更适用于大多数“极客玩家”的同时又兼容了实用性，安全性，以及可玩性的最简单的方案。当然，教程是模块化分离的，如果你是“复刻”的玩家我相信完全可以自己从教程中选取自己需要的部分来定制属于自己的网盘
